# Commit & Branch Policy

Canonical copy. `.claude/scripts/orchestrate-block.prompt.md` §G.1 is a
pointer into this file.

## Never push

**No agent ever runs `git push` under any circumstance.** Pushing to a
remote is a human action, performed by the founder from their own
terminal. This applies to every variant: `git push`, `git push origin`,
`git push --force`, `git push -f`, `git push --all`, `git push --tags`,
`git push -u`. It applies whether the branch is a feature branch, a
worker branch, `main`, or anything else.

This rule is enforced three ways:

1. `.claude/settings.json` denies every `Bash(git push*)` variant at the
   harness level: the tool call is rejected before it reaches the
   shell.
2. `.claude/rules/commit-policy.md` (this file) states the rule for
   humans and subagents that read rules but do not trigger the deny
   list.
3. Subagent prompts inherit this rule; pass it through when delegating.

If you think you need to push (CI, release automation, pre-deploy
sanity), the answer is still no. Ask the founder to push. The cost of
waiting for a human to push is always lower than the cost of an unwanted
push.

The same principle applies to `git pull`, `git fetch`, and
`git remote set-url`: also denied in `settings.json`. Agents work
against the local state they already have; the founder synchronises
with the remote.

## Conventional Commits

Every commit subject matches `<type>(<scope>): <subject>` where:

- `type` is one of: `feat | fix | refactor | test | docs | chore | build | ci | perf | style | revert`.
- `scope` is a project area. Aevum reserves a few core scopes that
  ship with the harness; projects extend the list at adoption time.
  - Core scopes (Aevum): `claude` (harness changes under `.claude/`),
    `scripts` (harness scripts), `docs`, `infra`.
  - Project scopes: every project adds its own (e.g. `core`, `api`,
    `ui`, plus one per domain subdir). Document the project's scope
    vocabulary in this rule at adoption time.
- `subject` is lowercase, imperative, no longer than 72 chars, no
  trailing period.

Example commit subjects:

- `feat(scripts): add baseline-diff JSON schema bump`
- `refactor(claude): split commit-policy hook into shared helpers`
- `fix(claude): tighten worker-worktree jail path-traversal check`
- `docs: add swap-points reference

## Forbidden in commit messages

- **AI authorship trailers.** No `Co-Authored-By: Claude` line, no
  "Generated with Claude Code" line, no robot-emoji "Generated"
  trailer, no "Generated by assistant" line. This applies to every
  commit from every agent; pass the rule through to subagents.

- **Non-ASCII in the subject line.** Full Unicode is allowed in the
  body. The subject is ASCII-only. Em-dash (Unicode U+2014) is
  forbidden in subjects AND bodies: use periods, semicolons, commas,
  parens, or rephrase. The commit-policy hook enforces both checks.
- **Empty subjects** or subjects with placeholders like `WIP`, `tmp`,
  `fix` (without scope and description).

## Forbidden git flags and commands

- `git push …` in any form: see §"Never push" above.
- `git commit --amend` / `git commit -C` on shared history: pre-commit
  hook failure means the commit did **not** happen; fix, re-stage,
  create a **new** commit.
- `git add -A` / `git add --all` / `git add .`: stage explicit paths.
  No exceptions, including the orphan-autosave branch in
  `.claude/scripts/orchestrate-block.sh` (when wired). The autosave
  preserves uncommitted work on a doomed worker branch when a wedge is
  detected, but it uses an explicit-paths pipeline: enumerate via
  `git ls-files --modified --deleted --others --exclude-standard`,
  drop any path matching gitignore via `git check-ignore --stdin
  --no-index`, restrict to a fixed allow-list of productive-work
  prefixes (`src/`, `docs/`, `scripts/`, `tests/`, `.claude/scripts/`,
  `.claude/agents/`, `public/`), and stage each surviving path with
  `git add -- <path>`. If no path survives, the autosave emits an
  `autosave_empty` ledger event and skips the commit. Settings-level
  secret-file blocks (`.env*`, `*.key`, `*.pem`, `id_rsa`, etc.)
  remain in effect as a second line of defence.
- `git reset --hard`, `git clean -fd`, `git rebase -i`, `git branch -D`,
  `git filter-branch`, `git filter-repo`, `git update-ref`,
  `git reflog expire`, `git gc --prune`.
- `git checkout --orphan`, `git checkout --detach`.
- Any `--no-verify` flag that skips hooks.

Destructive operations allowed only when the user explicitly requests
them.

## Branch naming

- `main` is the single working branch for direct, non-orchestrated work.
  No direct commits to `main` outside fast-forward merges of approved
  worker or block-integration branches.
- Block-orchestrated work:
  - `block/<BLOCK_ID>/integration`: persistent per-block integration
    branch.
  - `block/<BLOCK_ID>/<cluster_id>`: cluster branch.
  - `block/<BLOCK_ID>/<cluster_id>-<task_id>`: worker branch. Dash,
    not slash, at the last hop because git refs are hierarchical and
    cannot hold both `block/<BLOCK_ID>/<cluster_id>` and
    `block/<BLOCK_ID>/<cluster_id>/<task_id>` at the same time.
- Feature work outside the block model:
  `feature/<area>/<short-desc>`, for example
  `feature/api/auth-refresh`.

## Block-era merge subjects

When the block orchestrator merges branches it uses plain `chore` type:

- Task → cluster: `chore(block-<ID>): merge <cluster_id>-<task_id>`.
- Cluster → integration: `chore(block-<ID>): merge <cluster_id>`.
- Debt-paydown baseline refresh:
  `chore(block-<ID>): baseline refresh after <cluster_id>`.

`merge(<scope>): ...` as a type is forbidden (not in the
Conventional-Commits enum) and is rejected by
`.claude/hooks/commit-policy.sh`.

## Staging discipline

- Stage only the paths you touched. Never `git add .`.
- Never commit files likely to contain secrets: `.env*`,
  `credentials*`, `*.key`, `*.pem`, `*.p12`, `*.pfx`, `id_rsa`,
  `id_ed25519`.
- Never commit generated artefacts that belong in `.gitignore`. The
  set is language-specific (for example `node_modules/`, `.next/`,
  `tsconfig.tsbuildinfo` for Node; `__pycache__/`, `.venv/`,
  `dist/` for Python; `target/` for Rust). Project-specific patterns
  belong in the project's `.gitignore`.

## What the hook enforces

`.claude/hooks/commit-policy.sh` blocks:
- Missing or malformed Conventional-Commits prefix.
- AI-authorship trailers anywhere in the message
  (`Co-Authored-By: Claude`, "Generated with Claude Code",
  robot-emoji "Generated" trailers).
- `--amend`, `--no-verify`, `-A`, `--all` in the command line.
- Any non-ASCII character in the subject line (em-dash included; the
  hook does a strict ASCII check via grep + Python).

The hook runs at `PreToolUse` for `Bash` commands matching `git commit`.
